Те, что с черточкой хардкодятся на стороне сервера и не передаются.
Пользователь выбирает на сайте:

Для рассчета базового тарифа.

1. (ts_type_id) Тип ТС Id из справочника (Грузовик, Легковая, Мотоцикл и т.д.) 
	Выбор этого Id определяет значения для справочника марок ТС.
2. (ts_make_id) Марку ТС Id из справочника (Honda, Yamaha, Audi и т.д.)
    Выбор этого Id определяет значения для справочника Моделей ТС
3. (ts_model_id) Модель ТС Id из справочника (CX5, Priora, и т.д.)
	Выбор этого Id определяет значения для справочника Модификаций ТС
4. (ts_modification_id) Модификация TC Id из спраочника (Hatchback, Forester SX и т.д.)
Вышеперечисленные параметры отражаются в значение из следующего справочника.
Значение из этого справочника может быть выбрано и вручную, при остутсвии в базе данных по маркам-моделям
5. (ts_group_id) Группа ТС по квалификации страховой компании. Их собственная группировка. Id из справочника (Иномарки, Наши ведра, Газели и т.д.)
	Определяет доступные программы страхования и предельный срок эксплуатации ТС (сколько лет бегает с момента первой регистрации ГИБДД).
6. (damage_det_type_id) Тип определения возмещения при наступлении страхового случая
	Определяет доступные программы страхования и предельный срок эксплуатации ТС (сколько лет бегает с момента первой регистрации ГИБДД).
7. (ts_age) Срок эксплуатации ТС (сколько лет бегает с момента первой регистрации ГИБДД)
8. (ts_sum) Оценочная стоимость ТС, она же страховая сумма (та самая сумма на которую осущетсвляется страхование и которая выплачивается в случае полной гибели или угона)
9. (tariff_program_id) Программа страхования Id из справочника, у каждой страховой свои программы.
10.(risk_id) Набор рисков по которому осуществляется страховое покрытие, у каждой страховой могут быть свои пакеты. Никто не мешает делать вырожденные пакеты с одним риском для случая "конструкторов" тарифов
11. (amortisation) Признак учета амортизации при выплатах по страховым случаем, связанным с ущербом имеет смысл только для тарифа УНИВЕРСАЛ


Для рассчета поправочных коэфициентов:
12. (payments_without_references_id) Вариант выплат компенсации без предоставления справок Id из справочника. Каждая страховая сама определяет список таких вариантов.
13. (franchise_type_id) Тип франшизы при страховании Id из справочника. (Условная, Безусловная)
14. (contract_day) (contract_month) (contract_year) Длительность договора страхования.  Число, которое определяется из справочника. Причем какое поле будет присылаться зависит от значений в справочнике.
Особенность данных полей в том, что контракт может исчисляться в днях, может в месяцах, может в годах. (в нашем случае есть вариант в днях, остальные в месяцах)
15. (drivers_count) Количество лиц, допущеных к управлению ТС
16. (driver_age) Возраст самого старшего водителя среди лиц допущеных к управлению ТС
17. (driver_exp) Водительский стаж самого неопытного среди лиц допущенных к управлению ТС
18. (ts_no_defend_flag) Признак отсутсвтия каких либо дополнительных противоугонных средств на ТС
19. (ts_satellite_flag) Признак установленной на ТС спутниковой сигнализации (Сигналка с телеметрией)
20. (ts_have_electronic_alarm) Признак установленной на ТС обычной сигналки с обратной связью
- 21. (is_onetime_payment) Оплата полиса единовременно. Может быть false если полис еще и в рассрочку берется
- 22. (car_quantity) Количество машин, страхуемых одновременно (в парке)
23. (franchise_percent) процент от страховой суммы, 
которую страхователь не получит в случае наступления страхового случая (безусловная франшиза) или при определенных условиях получит (условная франшиза)
наличие данного параметра автоматически включает рассчет коэфициента франшизы
24. (commercial_carting_flag) Признак того, что ТС сдается в прокат или состоит в парке такси
- 25. (commission_percent) Процент комиссионного вознаграждения от суммы стоимости полиса
- 26. (is_legal_entity) Признак страхователя, Юридическое лицо
